# Contributing to USTF

**Status:** Draft

USTF uses a source-first, versioned contribution workflow.

## Contribution Types

- Core rule proposal
- Reference procedure improvement
- Genre module addition
- Template improvement
- Playtest report
- Typo or clarity fix

## Required Process

1. Identify affected files.
2. State whether the change affects Core, Module, Reference, Template, or Documentation.
3. Explain the reason for change.
4. Record compatibility impact.
5. Provide playtest evidence when applicable.
6. Use the Rule Change Proposal template for rule changes.

Core changes require stronger justification than module or template changes.
